🚨 Problemas Críticos Encontrados

❌

CRÍTICO #1 — SQUARE_CLIENT_SECRET tiene el valor incorrecto

El valor actual de SQUARE_CLIENT_SECRET es idéntico a SQUARE_ACCESS_TOKEN. Un access token (EAAAl...) NO es el client secret de OAuth. El OAuth client secret del sandbox tiene el formato sandbox-sq0csb-... y se obtiene en la consola de Square Developer → tu app → pestaña OAuth.

  • Mientras esté mal, el botón "Connect with Square" siempre fallará con token_exchange_failed
  • El flujo OAuth completo está bloqueado por este único error
❌

CRÍTICO #2 — NEXT_PUBLIC_SITE_URL apunta a localhost en Vercel

NEXT_PUBLIC_SITE_URL=http://localhost:3000 en .env.local es correcto para desarrollo local, pero en Vercel este valor se usa para construir la Redirect URI: http://localhost:3000/api/square/oauth/callback. Square rechazará este redirect porque no coincide con el dominio autorizado. Vercel necesita su propia variable de entorno con el dominio real.

  • Registrar Redirect URL para Vercel
    En la sección OAuth, agregar:
    https://reyco-barber-c6dm.vercel.app/api/square/oauth/callback
    ▶ Dónde: tu app → pestaña OAuth → "Redirect URL"
  • Registrar Redirect URL para localhost
    Agregar para desarrollo local:
    http://localhost:3000/api/square/oauth/callback
    ▶ Dónde: tu app → pestaña OAuth → "Redirect URL"
  • Verificar Application ID del sandbox
    Pestaña Credentials → pestaña "Sandbox" → "Application ID"
    Si tiene prefijo sandbox-sq0idb-…, ese es el valor correcto para NEXT_PUBLIC_SQUARE_APP_ID
    ▶ Dónde: tu app → pestaña Credentials → Sandbox
  • Confirmar OAuth Permissions (scopes)
    Verificar que estos 4 scopes estén habilitados:
    PAYMENTS_WRITE   PAYMENTS_READ   MERCHANT_PROFILE_READ   ITEMS_READ
    ▶ Dónde: tu app → pestaña OAuth → "OAuth Permissions"

Evaluación completa para integrar Square en SAASbarberiaSquare con soporte de pagos desde dispositivos móviles

✅

Factible — Arquitectura Compatible

El stack actual (Next.js 16 App Router · Server Actions · Supabase · TypeScript) es totalmente compatible con la Square Web Payments SDK y la Payments API. No se requiere refactorización arquitectural. La única fricción técnica real está en el CSP existente y la verificación de dominio para Apple Pay.

3. Conflicto Crítico: Content Security Policy

Tu proxy.ts implementa un CSP estricto con nonce. Square necesita dominios adicionales. Este es el cambio exacto que debes hacer:

CSP Actual en proxy.ts (BLOQUEARÁ Square)

function buildCsp(nonce: string): string {
  const isDev = process.env.NODE_ENV === 'development'
  return [
    "default-src 'self'",
    isDev
      ? `script-src 'self' 'nonce-${nonce}' 'strict-dynamic' 'unsafe-eval'`
      : `script-src 'self' 'nonce-${nonce}' 'strict-dynamic'`,
    "style-src 'self' 'unsafe-inline'",            // ✅ OK para Square
    `connect-src 'self' https://${SUPABASE_HOST} wss://${SUPABASE_HOST}`, // ❌ falta Square
    "frame-src https://www.google.com https://maps.google.com",  // ❌ falta Square
    ...
  ]
}

CSP Actualizado (compatible con Square)

function buildCsp(nonce: string): string {
  const isDev = process.env.NODE_ENV === 'development'
  const squareCdn = isDev
    ? 'https://sandbox.web.squarecdn.com'
    : 'https://web.squarecdn.com'
  const squarePci = isDev
    ? 'https://pci-connect.squareupsandbox.com'
    : 'https://pci-connect.squareup.com'

  return [
    "default-src 'self'",
    isDev
      ? `script-src 'self' 'nonce-${nonce}' 'strict-dynamic' 'unsafe-eval' ${squareCdn}`
      : `script-src 'self' 'nonce-${nonce}' 'strict-dynamic' ${squareCdn}`,
    "style-src 'self' 'unsafe-inline'",
    `img-src 'self' data: blob: https://images.unsplash.com https://${SUPABASE_HOST} https://api.dicebear.com`,
    `font-src 'self' data: https://square-fonts-production-f.squarecdn.com https://d1g145x70srn7h.cloudfront.net`,
    `connect-src 'self' https://${SUPABASE_HOST} wss://${SUPABASE_HOST} ${squareCdn} ${squarePci} https://o160250.ingest.sentry.io`,
    `frame-src https://www.google.com https://maps.google.com ${squareCdn}`,
    "frame-ancestors 'self'",
    "object-src 'none'",
    "base-uri 'self'",
  ].join('; ')
}
⚠️ Nota sobre strict-dynamic: Con 'strict-dynamic', los scripts cargados con nonce pueden cargar sub-scripts dinámicamente. Sin embargo, Square carga iframes para el formulario de tarjeta — por eso frame-src debe incluir el CDN de Square explícitamente.

4. Pasos de Implementación Completos

Crear cuenta en Square Developer Configuración
Ir a developer.squareup.com → crear aplicación → obtener Application ID, Location ID y Access Token del Sandbox. En producción se usan las credenciales del panel real.
Agregar variables de entorno Configuración
Añadir a .env.local las 4 variables de Square detalladas en la sección 5 de este documento. Las variables NEXT_PUBLIC_ son seguras para el cliente; las demás solo van al servidor.
Instalar Square Node.js SDK Dev
npm install square — el SDK oficial de Square para Node.js incluye tipos TypeScript nativos y el cliente para llamar a CreatePayment, RefundPayment, etc.
Actualizar CSP en proxy.ts Crítico
Agregar los dominios de Square a script-src, frame-src, connect-src y font-src. Sin este paso, el SDK queda bloqueado por el navegador.
Crear tabla payments en Supabase Base de datos
Nueva migración con la tabla payments (ver schema en sección 6), políticas RLS y foreign key a appointments. Agregar campo payment_status en appointments.
Crear app/actions/payments.ts Dev
Server action que recibe el token generado por el SDK del cliente, llama a square.paymentsApi.createPayment() y guarda el resultado en Supabase. Todo corre en el servidor — el Access Token nunca llega al cliente.
Crear componente PaymentForm Dev
Componente "use client" que carga el Web Payments SDK vía CDN, inicializa la card, Apple Pay y Google Pay, y llama al server action con el token. Se integra como un paso adicional en el BookingWizard existente.
Archivo de verificación Apple Pay Crítico para Apple Pay
Colocar el archivo descargado desde el Developer Console en public/.well-known/apple-developer-merchantid-domain-association sin extensión de archivo. Requiere dominio con HTTPS real (no localhost).
Configurar Webhooks para actualizaciones de pago Dev
Crear una ruta API (app/api/square-webhook/route.ts) que reciba eventos de Square (payment.completed, payment.failed) y actualice el estado en Supabase. Verificar la firma HMAC con SQUARE_WEBHOOK_SIGNATURE_KEY.
Testing en Sandbox → Deploy a producción QA
Usar las tarjetas de prueba de Square (4111 1111 1111 1111). En producción, cambiar las env vars al dominio web.squarecdn.com y las credenciales reales de Square. Registrar el dominio en el Developer Console.

5. Variables de Entorno Necesarias

Variable Scope Descripción Dónde obtener
NEXT_PUBLIC_SQUARE_APP_ID Público Application ID del proyecto Square. Seguro en el cliente — inicializa el Web Payments SDK. Developer Console → tu aplicación → Credentials
NEXT_PUBLIC_SQUARE_LOCATION_ID Público ID de la ubicación (barbería). Requerido para inicializar pagos y Apple Pay. Developer Console → tu aplicación → Locations
SQUARE_ACCESS_TOKEN Secreto Token del servidor para llamar a la Payments API. Nunca exponerlo al cliente. Developer Console → tu aplicación → Credentials → Access Token
SQUARE_WEBHOOK_SIGNATURE_KEY Secreto Clave HMAC para verificar que los webhooks vienen realmente de Square. Developer Console → Webhooks → tu endpoint → Signature Key
ℹ️ En Sandbox, los valores de NEXT_PUBLIC_SQUARE_APP_ID empiezan con sandbox-sq0idb-. En producción empiezan con sq0idp-. Hay que cambiar ambas variables al pasar a producción.

6. Schema de Base de Datos a Agregar

-- Nueva tabla de pagos
CREATE TABLE public.payments (
  id                  uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  appointment_id      uuid REFERENCES public.appointments(id) ON DELETE CASCADE NOT NULL,
  user_id             uuid REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL,
  square_payment_id   text UNIQUE NOT NULL,               -- ID devuelto por Square
  amount_cents        integer NOT NULL,                  -- Square trabaja en centavos
  currency            text DEFAULT 'USD' NOT NULL,
  status              text DEFAULT 'pending'
                        CHECK (status IN ('pending', 'completed', 'failed', 'refunded')),
  payment_method      text,                                   -- 'card', 'apple_pay', 'google_pay'
  card_last4          text,                                   -- últimos 4 dígitos (solo referencia)
  created_at          timestamp with time zone DEFAULT now() NOT NULL,
  updated_at          timestamp with time zone DEFAULT now() NOT NULL
);

-- Agregar payment_status a appointments
ALTER TABLE public.appointments
  ADD COLUMN payment_status text DEFAULT 'unpaid'
    CHECK (payment_status IN ('unpaid', 'paid', 'refunded'));

-- RLS: clientes solo ven sus pagos, admin ve todos
ALTER TABLE public.payments ENABLE ROW LEVEL SECURITY;

CREATE POLICY payments_select ON public.payments
  FOR SELECT USING (auth.uid() = user_id OR is_admin());

CREATE POLICY payments_insert ON public.payments
  FOR INSERT WITH CHECK (auth.uid() = user_id);

7. Flujo de Pago Completo (Móvil)

// ① Cliente toca "Pagar con Apple Pay / Google Pay" en el móvil

// ② Web Payments SDK muestra el sheet nativo del dispositivo
const tokenResult = await applePay.tokenize()
//    token: "cnon:CBASENab2qIGo4..."  (válido por 24 horas)

// ③ Server action recibe el token (servidor, nunca el cliente maneja el token)
'use server'
async function processPayment(token: string, appointmentId: string) {
  const { result } = await squareClient.paymentsApi.createPayment({
    sourceId: token,
    idempotencyKey: crypto.randomUUID(),
    amountMoney: { amount: 5000n, currency: 'USD' },   // BigInt — Square usa centavos
    locationId: process.env.NEXT_PUBLIC_SQUARE_LOCATION_ID!,
  })

  // ④ Guardar resultado en Supabase
  await supabase.from('payments').insert({
    appointment_id: appointmentId,
    square_payment_id: result.payment!.id,
    amount_cents: 5000,
    status: 'completed',
    payment_method: result.payment!.sourceType,
  })

  // ⑤ Actualizar estado del appointment
  await supabase.from('appointments')
    .update({ payment_status: 'paid' })
    .eq('id', appointmentId)
}

9. Riesgos a Considerar

🚨
Square no opera en todos los países. Verificar en squareup.com/global si tu país está soportado antes de iniciar. Si la barbería está en Latinoamérica (excepto México), Square no está disponible — evaluar Stripe como alternativa.
⚠️
Apple Pay no funciona en localhost. Para probar Apple Pay en desarrollo necesitarás un dominio HTTPS real (puede ser un tunnel con ngrok o un subdominio de prueba). Google Pay sí funciona en localhost con las credenciales sandbox.
⚠️
Square trabaja con BigInt en Node.js. Los montos de dinero (amount) son bigint en el SDK de TypeScript. Hay que serializar a number antes de guardar en Supabase o devolver al cliente.
ℹ️
Pago al momento de la reserva vs. pago al llegar. Debes decidir el modelo de negocio: ¿cobrar 100% al reservar, depósito parcial, o solo al finalizar el corte? Square soporta los tres modos (capture inmediata, autorización diferida, o pago manual desde el admin).
